## protocol

- simplify: reduce the problem to its essential form
- learn from analogues: study similar problems, trace paths to known solutions
- approach from different angles: rotate perspective, try the inverse
- decompose: break into atomic parts that can be solved independently
- solve backwards: start from the desired solution, work toward the problem
- extend: push solutions as far as they can go, test their limits

## application

every agent session should internalize this protocol. when stuck, cycle through these steps rather than repeating the same approach. when planning, use decompose + solve backwards to map the work. when reviewing, use extend to find edge cases

the methodology is embedded in the `shannon-methodology` qdrant collection for semantic retrieval. agents can query it for worked examples as they accumulate
